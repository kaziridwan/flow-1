# FLOW-1 — Road to v1

## Context

FLOW-1 is a working browser-only Pomodoro focus instrument (Vite 6 · React 19 · TS · Tailwind v4, no backend). Today the audio system is a **single flat picker of 8 sources** (`AudioPicker.tsx`) backed by a **single-source engine** (`src/lib/audio.ts`) that can play one of: noise (white/pink/brown), binaural (3 fixed presets), or an external URL (YouTube/podcast/media).

Two problems drive this milestone:

1. **The Sound panel is overcrowded** — 8 equal-weight buttons + conditional sub-panels create cognitive load (violates the project's "one bold element, everything else quiet" rule, ADR-6).
2. **The audio vision has outgrown the engine** — we want a *Noise Designer* (Endel-style X/Y blend + low-pass) and a *Binaural Engine* (keyframed tracks), neither of which the single-source, no-automation engine supports.

**Outcome of v1:** a calmer, categorized Sound panel (3 categories), two real sound-design surfaces, and the reliability hardening that makes the app release-grade. Each phase below is **independently shippable**. (Source of truth for the audio vision: `agents-context/plans/20260617091148-road-to-v1.md`.)

**Decisions taken with the user:**
- Sound panel = **3 categories** (Noise · Binaural · Custom Media). **Mixes is deferred to v1.x** (Phase 6), but the engine is built mixer-ready so it slots in cleanly.
- v1 = **audio redesign + reliability hardening**.
- Delivery = **incremental** — ship at the end of each phase.

## Target data model

Replace the flat `AudioSource` union in `src/types.ts:35-53` with a **category + per-category config**, versioned for migration.

```ts
type AudioCategory = "none" | "noise" | "binaural" | "media";

interface AudioConfig {
  v: 2;                       // schema version (drives migration)
  category: AudioCategory;
  noise: NoiseDesign;
  binaural: BinauralDesign;
  media: { kind: "youtube" | "podcast" | "url"; url: string };
  volume: number;            // master 0..1
  pauseOnBreak: boolean;
}

interface NoiseDesign {       // Phase 2
  blend: { x: number; y: number };          // 0..1 each → corner weights
  lowpass: { enabled: boolean; cutoff: number; q: number };
  volume: number;
}

interface BinauralDesign {    // Phase 3
  durationSec: number;
  keyframes: { t: number; base: number; beat: number; volume: number }[];
  // invariant: sorted by t, first t=0, last t=durationSec
}
```

This shape is **forward-compatible with Mixes**: a future `mix: { layers: {...}[] }` is just one more category.

> Implementation note: the v2 root was named `AudioSettings` (the legacy `AudioConfig`/`AudioSource` types were removed) to avoid a confusing rename during the incremental migration. Everything else matches this model.

## Phase 0 — Test harness + config migration (foundation, shippable) ✅ DONE

*Low risk, unblocks every schema change below.*

- Add **Vitest** (devDep) + `test` script. Cover the pure logic that already exists: `src/lib/schedule.ts` (meal placement, long-break-after-4th rule, no trailing break on last focus) and `src/lib/format.ts` (`youtubeId` URL parsing). Highest-ROI tests; no UI harness needed.
- Replace the shallow `load()` merge (`src/App.tsx:30-37`) with a **versioned `migrateAudio()`** in a new `src/lib/migrate.ts`. v1→v2 mapping: `white|pink|brown → noise` (blend pinned to that corner), `binaural → binaural` (two flat keyframes from the old preset), `youtube|podcast|media → media`, `none → none`. Add unit tests for the migration.
- **Ship:** no user-visible change; safety net + schema versioning in place.

## Phase 1 — Categorized Sound panel (shippable) ✅ DONE

Collapse 8 flat buttons into **Silent + 3 category cards** in `AudioPicker.tsx`, reusing `Segmented`, `Eyebrow`, `Toggle` from `src/components/controls.tsx` and the neumorphic utilities.

- **Noise** card: shows current default inline (e.g. "White"); a **⋯ menu** opens the Noise Designer (Phase 2 — stub to "coming soon" / default until then).
- **Binaural** card: shows current default (e.g. "Flow"); **⋯ menu** opens the Binaural Engine (Phase 3 — stub until then).
- **Custom Media** card: expands to sub-tabs **YouTube · Podcast · Media URL** (reuse the existing URL input + `youtubeId` validation from `format.ts`).
- Rework `AudioController.tsx` routing to switch on `cfg.category` instead of the flat source.
- Add a lightweight **modal/sheet** primitive (none exists today) for the designers; keep it neumorphic and quiet (ADR-6), with focus-trap + `aria-label` (accessibility floor).
- **Key files:** `src/components/AudioPicker.tsx`, `src/components/AudioController.tsx`, `src/types.ts`, new `src/components/Sheet.tsx`.
- **Ship:** the calmer 3-category panel; media fully works; Noise/Binaural use existing defaults.

## Phase 2 — Noise Designer (shippable) ✅ DONE

Endel-style **X/Y blend pad + low-pass filter**, opened from the Noise card's ⋯ menu.

- **Engine:** extend `AudioEngine` to play a *blend* of noise BufferSources simultaneously through per-source `GainNode`s summed into one `BiquadFilter` (`type:"lowpass"`, cutoff + Q) → design gain → master. Reuse `buildNoiseBuffer()` (`audio.ts:71-101`). This mini-mixer is the seed for Phase 6 Mixes.
- **Corners:** rectangle's 4 corners = **White · Pink · Brown · Blue** (blue generator already added in Phase 1). Pad position → **bilinear corner weights** (`cornerWeights` in `audioDesign.ts`) → per-source gains. *(4th color is adjustable; 3 corners on a triangle is the fallback if blue noise proves not worth the DSP.)*
- **UI:** draggable square pad (a new `XYPad` component; respect `prefers-reduced-motion`), plus a low-pass toggle with cutoff + volume sliders (reuse the slider pattern in `AudioPicker.tsx`).
- **Persist** `NoiseDesign`; the Noise card reflects the custom blend.
- **Key files:** `src/lib/audio.ts`, new `src/components/NoiseDesigner.tsx`, new `src/components/XYPad.tsx`.
- **Ship:** users can tune and save a custom noise. Replaces the interim dominant-corner playback.

## Phase 3 — Binaural Engine (shippable) ✅ DONE

Design a **binaural track of a chosen length** with **interpolated keyframes**, opened from the Binaural card's ⋯ menu.

- **Engine:** add `playBinauralTrack(design)` building on the existing two-oscillator + `ChannelMerger` topology (`audio.ts`). Schedule `left.frequency`=base and `right.frequency`=base+beat plus gain via `linearRampToValueAtTime` across keyframe times; **loop** by re-anchoring the schedule at `ctx.currentTime` on track end. (This is the live-retarget hook the Roadmap names for future HR-adaptive beats.)
- **Defaults:** a fresh design seeds two keyframes (start & end) from a base/beat/volume — matching the markdown's "default acts as keyframes at start and end."
- **UI:** a keyframe **timeline** — add/drag keyframes at a time `t`, each editing base freq, beat (frequency difference), and volume; show the interpolated curve. Keep v1 modest (numeric inputs + simple track viz) rather than a full DAW.
- **Persist** `BinauralDesign`; legacy presets already migrate into seed designs (Phase 0).
- **Key files:** `src/lib/audio.ts`, new `src/components/BinauralEngine.tsx`.
- **Ship:** users can design evolving binaural tracks. Replaces the interim flat first-keyframe playback.

## Phase 4 — Reliability & UX hardening (shippable) ✅ DONE

The "release-grade" items from the existing roadmap, independent of audio:

- **`aria-live`** announcements on the timer + keyboard shortcuts (space = pause/resume, n = skip) — `RunScreen.tsx` / `useTimer.ts`.
- **Persist an in-progress run** across reloads (store `committed` plan + elapsed; restore on load) — `App.tsx`.
- **Crossfade** noise/binaural on break transitions instead of hard restart (closes a known tech-debt gap) — `AudioController.tsx` / `audio.ts`.
- **Ship:** smoother, more accessible run experience.

## Phase 6 — Mixes *(v1.x, deferred)*

Multi-source mixer with mynoise-style sliders blending any prior source (a noise preset, a binaural design, etc.). Build directly on the Phase 2 mini-mixer: each layer = a source + gain into a master bus. Add a `mix` category + `MixDesign` to the model. **Out of v1 scope** — documented here so the engine work above stays mixer-ready.

## Reuse (don't rebuild)

- **Noise generation:** `buildNoiseBuffer()` — `src/lib/audio.ts`.
- **Binaural topology:** two oscillators + `ChannelMerger` — `src/lib/audio.ts`.
- **Gain-envelope pattern:** the bell's `setValueAtTime`/`exponentialRampToValueAtTime` — `src/lib/audio.ts`.
- **Pure audio-design helpers:** corners, presets, defaults, `cornerWeights`, `dominantColor` — `src/lib/audioDesign.ts`.
- **UI primitives:** `Segmented`, `Eyebrow`, `Toggle`, slider markup — `src/components/controls.tsx` + `AudioPicker.tsx`; modal `Sheet` — `src/components/Sheet.tsx`.
- **URL parsing:** `youtubeId()` — `src/lib/format.ts`.
- **Design tokens / neumorphic utilities:** `@theme` + `@utility` in `src/index.css`. Add colors there; don't hard-code hexes. Preserve ADR-6 (one bold element) and the accessibility floor (focus rings, `aria-label`, `prefers-reduced-motion`).

## Verification

- **Gate (every phase):** `pnpm run build` must pass (strict TS — `noUnusedLocals`, `verbatimModuleSyntax`, type-only imports).
- **Tests:** `pnpm test` (Vitest) green for `schedule`, `format`, and `migrate`.
- **Manual (`pnpm run dev`) per phase:**
  - P1: each category selects/plays; media tabs accept URLs; ⋯ opens the sheet; reload restores via migration; old localStorage upgrades cleanly.
  - P2: dragging the X/Y pad audibly shifts the blend; low-pass cutoff/volume audibly change; design persists across reload.
  - P3: a multi-keyframe track audibly sweeps base/beat/volume over time and loops; design persists.
  - P4: screen reader announces phase changes; space/n shortcuts work; run survives reload; no audible gap on break transitions.
